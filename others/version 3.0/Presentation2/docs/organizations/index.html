Search Bar and Filter Bar Help Menu

Search Bar Usage

Description

The search bar provides the ability to quickly navigate to an entity in the listing.

Usage

In the "Search for entities" bar, begin typing the name of an entity of interest.

A dropdown will display suggestions based upon the text in the search bar.

To select the entity from the dropdown, use the down arrow key to navigate to the particular entity, then press tab for the selected entity to fill in the text field.

Finally, press enter and the entity list will jump to the selected entity. To view that entitiy's documentation, press enter again or click on the entity in the list

Note: Queries with the search bar are case insensitive.

Filter Bar Usage

Description

The filter bar provides the ability to filter the list of entities by attribute.

Usage

The mechanics of using the filter bar are the same as the search bar mechanics described above.

The filter bar supports complex filters using the keywords "and", "or" and "not" and parentheses "(" and ")".

Try the example filter below: 

 ((startOn or endOn) and metadata)

Note: Queries with the search bar are case sensitive.

Overview

Description

In HEDM, the organizations entity is used to define collections of people organized into a structure that acts as a single agent. Organizations can represent the institution itself or any of the various stuctures within an institution (colleges, departments, ...). Businesses can also be represented as organizations. Since organizations can represent a wide range of structures, they can fill various roles when interacting with higher education (e.g. degree granting institution, donor, vendor, ...). This entity contains only the information that is common to all organizations. Properties associated with roles organization may fill will be contained in role specific entities (e.g. constituents, vendors, ...) which will be developed and added to HEDM as required.

The organizations entity includes the following properties associated with the organization:

  • the type (e.g. 'college', 'department', 'business'),
  • a list of locations (i.e. addresses),
  • a list of phone numbers,
  • a list of email addresses,
  • a list of social media addresses.

The locations specified for a organization include the address lines, address type ('Main', 'P.O. Box', ...), the status of the address ('active', 'inactive') and if the address should be considered the 'Primary' address of the specified type. In addition, the location can include latitude and longitude and can be assigned to multiple geographic regions/sub-regions for reporting purposes. The mailing address for each location can also be specified in a country specific format.
